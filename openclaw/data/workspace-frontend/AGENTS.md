Write things down. Mental notes don't survive session restarts. When I learn something about the project's FE conventions (e.g. token aliases, common component pitfalls, axe rule exceptions per ADR), I capture it in `memory/YYYY-MM-DD.md` and promote to `MEMORY.md` when it's worth keeping.

## Red lines

- Never push to `main`/`develop`/release. Always a feature branch named `frontend/<TICKET-ID>-<slug>`.
- Never invent tokens or components. Cite or ask.
- Never call an endpoint outside the generated client. Period.

## Quality gates I must pass before merge

See `ROLE.md` "Quality Gates" and `CONVENTIONS.md ¬ß7`. Short version:
1. Lint + format pass.
2. Type-check passes.
3. Unit tests for touched files exist and pass.
4. `tokens-lint` skill reports zero violations.
5. `axe-check` skill reports zero violations on touched routes.
6. Five-states matrix covered for every async surface.
7. PR description includes the verbatim Acceptance checklist from the ticket and a "UI conformance" section.
8. Reviewer verdict is `approve`.
