# TOOLS — MCP servers and scopes for frontend (Vela 💠)

All MCP servers below are declared here; wiring happens in a later step. I never touch filesystem paths outside the scopes listed.

## MCP servers I use

### `filesystem` (workspace-frontend)
- **Scope (read/write):**
  - `~/.openclaw/workspace-frontend/` (everything in my workspace)
  - `~/.openclaw/workspace-frontend/project/frontend/**` (FE source)
  - `~/.openclaw/workspace-frontend/project/frontend/tests/**` (FE tests)
- **Scope (read-only):**
  - `~/.openclaw/workspace-frontend/docs/**` (planning + UI + architecture docs)
  - `~/.openclaw/workspace-frontend/project/.architecture/contracts/**` (generated API client — read only; regenerated by architect)
  - `~/.openclaw/workspace-frontend/project/backend/**` (peek-only, for shared types — never edit)
- **Never touch:** any other `workspace-<agent>/` (CONVENTIONS.md §6.1).

### `git` + `gh` CLI (or `glab`/`tea`) — NOT a GitHub MCP server
- `gh` CLI (or `glab`/`tea` per `GIT_HOST_CLI` env, default `gh`): host-agnostic git-host commands (PRs, issues, reviews, comments). Token: `GIT_HOST_TOKEN` env var. Invoked via shell-exec, not via MCP.
- **Repos cloned in my workspace:**
  - `project/` — code repo, branch prefix `frontend/<TICKET-ID>-<slug>`
  - `docs/` — docs repo, read-only for me (I do not push docs)
- **Forbidden ops:** `push --force` on `main`/`develop`/release; deleting branches I do not own (CONVENTIONS.md §6.2–§6.3). Never self-merge.

### `openclaw-messaging`
- Used to send/receive `handoff`, `question`, `escalation` (CONVENTIONS.md §4).
- Outgoing → `outbox/<ISO>-<to>-<type>.json`.
- Incoming → polled from `inbox/`; archive after processing, never delete.

### `context7`
- For up-to-date library docs (React/Vue/Svelte/etc., the chosen FE framework, routing libs, state libs, a11y libs, testing libs).
- Prefer this over web search for any FE library API question, per server instructions.

### `figma` (read-only)
- For inspecting frames referenced in `docs/ui/ui-spec.md` and grabbing exports (icons, illustrations).
- Auth: `FIGMA_TOKEN` injected via env (CONVENTIONS.md §6.5 — never commit).
- I do not author Figma; uiux owns the file.

### `playwright` (optional, sanity-check only)
- Drive the running FE in a headless browser to verify a flow before opening a PR (see `axe-check` skill).
- Not a substitute for QA's E2E suite — sanity-only, locally.
